- **s03 content error:** "FY16 through FY26 yard split" subtitle on KPI card #3 and BIW profile card both say FY26, but the count `11 BIW, 13 Ingalls` matches the wiki's DDG 127–150 total which runs through FY27 (DDG 150 is an FY27 Ingalls hull, per wiki ch 2 hull table). Pick one — either change subtitle to FY27 or trim the Ingalls count to 12.

## Key decisions and tradeoffs

- **Simplification level (aggressive).** Chosen by the user after seeing the dense first-pass slides. Each body slide is now one primary visual + one caption. Cost: lost the supporting data density (metric cards on s01, cost table on s02, hull matrix on s03). Benefit: each slide reads cleanly at glance.
- **Self-contained pipeline via slim-copy, not full copy.** Audited `build.py` for exactly what gets read from `_extracted/` and copied only those files. 164 KB vs the 1.9 MB it would have been for a full copy. ~90% reduction.
- **Pipeline left where the deck/ subfolder is the portable unit,** not the destroyer_outsourced_work/ parent. Edit a single `THIS_DIR / "_extracted"` line in build.py; zip and copy `deck/` anywhere.
- **Connector geometry on s02 ship map** — the original code computed `w = max(0.01, tx - cx_label)` which degenerated to a 0.01" stub for every label in `items` (since all targets sit left of label center). Rewrote to use proper bbox + `flipH` when needed. Doesn't change layout_check counts (connectors aren't flagged), but the diagonals are now visible at zoom rather than appearing as near-vertical hairlines.

## Known limitations carried forward

- **`\n` in network-node labels (now removed)** — the first-pass s01 used `"BIW\nBath"` literal newlines in run text. LibreOffice's PDF export interpreted these as line breaks, but PowerPoint may render the raw character. Not currently an issue since the network band was dropped in the simplification pass.
- **layout_check rect-containment filter** still suppresses real overlaps (the s02 ship-map labels were the canonical example). For future dense slides, expect to do a visual pass on top of layout_check.
- **Wiki content cross-check** caught two content issues on the first-pass s03 (FY26 vs FY27 subtitle inconsistency; FY26 silently skipped in the bar series). The current simplified s03 addresses the FY26 skip in its caption text but the original first-pass content errors are now moot.
