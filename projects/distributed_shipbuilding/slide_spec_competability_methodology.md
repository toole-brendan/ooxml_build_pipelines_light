# Slide Spec — Competable Work Identification Methodology

```text
SLIDE ## - COMPETABILITY METHODOLOGY
Type: Body

Slide's purpose:
  Sets up the next research phase. Defines how we will identify which work
  inside the already-sized outsourced Basic Construction pool is realistically
  competable for a new entrant, why the analysis runs at the existing
  work-type level rather than a new taxonomy, what structural signals the
  subaward record can evidence, and what the course of action is. This is a
  methodology slide in the same spirit as the work-type slide's methodology
  ledger: it documents an approach before results exist, so the audience can
  challenge the method now rather than the findings later.

TITLE
-----
Copy:
  Competability Methodology | Outsourced work is scored at the work-type
  level on five structural signals computed from the subaward record, with
  the vendor roster inside each work type naming the specific targets.

HIGH-LEVEL LAYOUT
-----------------
Two-zone body. LHS roughly two-thirds: the signal-definition table as the
main exhibit, with a horizontal four-step course-of-action strip directly
beneath it. RHS roughly one-third: a commentary column in the established
RHS-rail idiom, with a bordered exclusions-and-caveats callout box anchored
at the bottom of the rail, matching the bordered-note treatment used on the
work-type and penetration slides.

COMMENTARY COPY, IF NEEDED
--------------------------
Commentary 1
  Copy:
    Unit of analysis: the seven existing work types.
    Competability is reported per work type — the same mutually exclusive
    structural / machining / castings / piping / electrical / HVAC /
    coatings vocabulary already used for the pool sizing — so the competable
    pool nests inside exhibits the audience has already seen. All signals
    are computed per vendor and rolled up: the ranked vendor list inside
    each work type, with each vendor's award history, is itself the
    specific-award target list. No new taxonomy is introduced.
  Text / shape treatment:
    No-fill text rail, bold lead-in phrase then regular text, top of the
    RHS column under a bold "Methodology" zone header.

Commentary 2
  Copy:
    Why structure, not narrative: subaward descriptions are unusable.
    Prime filings carry boilerplate description text, so work content cannot
    be read from the award records. Classification therefore rides on the
    vendor — entity registry, NAICS, and name evidence — and competability
    rides on observable market structure: who shares the work, who is
    entering, how often the buy recurs.
  Text / shape treatment:
    Same rail treatment as Commentary 1, second block.

Commentary 3
  Copy:
    Demand-side overlay: where second sources are being actively seeded.
    Quantitative structure is crossed with stated intent — submarine
    industrial-base supplier-development announcements, prime supplier-day
    solicitations, and the PB27 Strategic Outsourcing narrative — to
    separate "fragmented because competed" from "single-source and the
    customer wants out."
  Text / shape treatment:
    Same rail treatment, third block.

OTHER VISIBLE COPY, IF NEEDED
-----------------------------
Copy:
  Exclusions-and-caveats callout, bottom of RHS rail:
    "Excluded from the competable pool: the ~$4.2B BlueForge Alliance
    subaward flow — supplier-development and workforce funding routed
    through the Columbia prime contract, not component purchases — and
    Navy-directed GFE primes, where the competition is run by the Navy,
    not the shipbuilder. First-tier subaward reporting is incomplete, so
    work-type pool sizes are floors; concentration, entry, and cadence
    survive undercounting better than levels do."

  Course-of-action strip, beneath the main table, four numbered steps
  reading left to right:
    1. Coverage — measure the unbucketed dollar share; extend the entity
       registry until the top recipients covering ~90% of dollars are
       classified. DDG pulls brought to parity with the submarine
       full-history corpus.
    2. Score — compute pool size, concentration, entry, and cadence per
       work type from the existing FY22–25 subaward corpus.
    3. Overlay — assign barrier height per work type and attach seeding
       evidence from industrial-base and prime sourcing announcements.
    4. Target — rank vendors within the highest-scoring work types; each
       vendor's award history yields the named competable awards.

Text / visual treatment:
  Callout: bordered box, 1pt black outline, no fill, centered italic text —
  the established open-question / method-note idiom. Course-of-action strip:
  four equal-width numbered stage cells in a single row, light fill with a
  bold step number and short bold verb, two to three lines of regular text
  per cell; visually subordinate to the main table above it. The strip is a
  plan, not a result — it should read as secondary to the signal table.

CHART, IF NEEDED
----------------
None. This is a method slide; the exhibit is the signal-definition table.
No placeholder chart should be added.

TABLE, IF NEEDED
----------------
Copy / rows / columns:
  Main exhibit. Five columns:
    # | Signal | What it measures | Data basis | Reads as competable when…
  Five rows:
    1 | Pool size | Outsourced dollars per work type, FY22–25 |
        Full-history first-tier subaward corpus, prime shipbuilding PIIDs |
        The work type is large enough to matter; small pools are deprioritized
        regardless of structure.
    2 | Supplier concentration | How many vendors share the work type and
        how unevenly, measured per vendor within each type |
        Subaward dollars grouped by vendor identity |
        Fragmented: the prime demonstrably multi-sources the work.
        Single-source reads as locked unless signal 5 shows the customer
        seeking a second source.
    3 | New-entrant rate | Vendors whose first-ever subaward lands in the
        most recent fiscal years, by work type |
        First-award date per vendor across the corpus |
        Recent entry is direct evidence the door is open; a widening entrant
        cohort marks where distributed-shipbuilding growth is actually landing.
    4 | Award cadence and churn | Repeat-order intervals per vendor; vendor
        exits offset by entries in the same work type |
        Award-date streams per vendor |
        Annual repeat buys are re-decided regularly; observed switching
        events prove the scope can move. One lifetime award reads as locked.
    5 | Barrier height and seeding | Certification burden — SUBSAFE,
        Level I material, nuclear-grade quality programs versus conventional
        fabrication — crossed with active second-source demand |
        Domain scoring per work type; industrial-base and prime sourcing
        announcements |
        Low certification barrier, or high barrier with active seeding —
        which signals a longer qualification path, not a closed door.

Purpose:
  Defines the scorecard before it is populated: what each signal is, where
  it comes from, and how to read it. The audience should leave able to
  predict what the populated scorecard will look like and to challenge any
  signal's logic now.

Structure:
  Method-definition table. Narrow # and Signal columns as the scan spine;
  "What it measures" and "Data basis" mid-width; "Reads as competable when…"
  widest, since the interpretation is the substance. Rows ordered from
  sizing to structure to overlay — the same order as the course-of-action
  strip below, so the two read as definition then execution.

Cell / table treatment:
  Header row bold on a light fill. # and Signal cells bold, top-left
  aligned; remaining cells regular weight, top-left, wrapped. Horizontal
  rules between rows only; no vertical rules except after the Signal
  column to set off the scan spine. In row 2 and row 4, the locked-reading
  sentence may be italic to distinguish the negative reading from the
  positive one.

Fit behavior:
  Shorten "What it measures" first, then "Data basis" — both can compress
  to fragments. The "Reads as competable when…" column must keep full
  sentences. If a row must compress hard, row 1 gives up text first; rows
  2, 3, and 5 carry the argument and must remain readable. Do not add a
  sixth signal row to fill space.

OBJECT NOTES, IF NEEDED
-----------------------
  - Reading order: title, signal table, course-of-action strip, RHS rail
    top to bottom, callout last.
  - The course-of-action strip must align its left and right edges to the
    signal table above it.
  - No purple working-flag chips on this slide; open items live in the
    bordered callout.
  - Leave the strip-to-table gap visibly smaller than the table-to-title
    gap so the strip reads as belonging to the table.

  solicitations.
```
