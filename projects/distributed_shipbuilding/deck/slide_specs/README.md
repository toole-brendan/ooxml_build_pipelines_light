# slide_specs

One markdown spec per slide (filename matches the slide module under
`deck_consolidated/slides/`). These are design notes for what each slide should
say and show — the build pipeline does **not** read them; they exist to plan and
review slides before/while you write the Python module.

See `deck_core/slide_guide.md` and `deck_core/slide_snippets.md` for the house
chrome/style conventions, and `deck_core/slide_base_template.py` for the starting
point of a new slide module.
