### 7. Policy reversal - cross-platform weapon system sustainment IN

User input: J014 is a services PSC (maintenance of guided missiles), not a procurement PSC. The workbook already includes weapons system maintenance services (J010 MK 45 gun M&R, K010 weapons modification, N010 weapons installation) for ship-integrated systems. The principle should be sharper than "same as aviation exclusion":

- **Ship-integrated weapon sustainment** (MK 45 gun, VLS launchers, CIWS, SLQ-32 EW, AEGIS radar, Trident launch tubes/fire control, Trident missile itself) -> IN the Services TAM
- **Cross-platform weapon system sustainment** on ship/sub platforms (Trident II D5 missile body, Tomahawk weapon maintenance) -> IN, they're Navy sustainment work whose platforms are ships
- **Aviation weapon sustainment** (AIM-9X, AMRAAM, Sparrow through NAVAIR / NAWC / FRC) -> OUT, aviation is a separate TAM
- **Weapon PRODUCT procurement** (PSC 14xx missile products, PSC 10xx gun products) -> OUT, and already lives on the Product Procurement sheet via `NEWBUILD_PSCS`

Reversed the SSP and Trident filters:

- Removed `STRATEGIC SYSTEMS PROGRAMS` from `SHORE_BASE_OFFICE_PATTERNS`
- Removed `NON_SHIP_MRO_DAP_PATTERNS` entirely and its check from `is_shore_base_excluded`
- Kept the NAVAIR / NAWC / FRC aviation office patterns (aviation stays out)

Net after reversal: kept $402.4M of new PSC $ (gained back $319M Trident + SHIPALT), excluded $117.4M (mostly aviation + SEAPORT-NXG).

Biggest returner: the $318.3M Draper MK7 LE2 contract is now correctly classified at the cascade's Tier 0c (DAP lookup) to `SSBN-726 Ohio` / Submarines supergroup / SSBN hull program (no manual override needed - the DAP "TRIDENT II MISSILE" maps to Ohio-class SSBN in `DAP_TO_VESSEL`).

### 11. Vessel Type vs Hull Program methodology call

User surfaced a reasonable concern: the Vessel Type cross-tab at the top of Services shows ~1.9% Unclassified, but the Hull Program cross-tabs below it show ~30% Unclassified. Investigated the cause:

- **Fully residual** (no supergroup, no hull): $128M = 1.9%
- **Supergroup-only** (classified to vessel type but not hull): $1,899M
  - `residual_override` (LLM + FPDS MOD 0 cascade): $1,100M
  - `supergroup_idv` (Tier 4 regex): $308M
  - `supergroup_mod` (Tier 3 regex): $289M
  - `recipient_prior` (Tier 5): $202M
- **Both supergroup and hull**: $4,767M

Root cause: 393 of the 403 LLM overrides set supergroup only (vessel_class empty), because the LLM prompt explicitly says "leave vessel_class null if you can't match one of these canonical labels." This is conservative by design - when the SOW says "SWFTS submarine combat systems support", the LLM correctly answers `supergroup=Submarines, vessel_class=null` because it can't tell Virginia vs Los Angeles vs Columbia vs Seawolf without more signal.

Decision: accept as the honest methodology floor. Hull Program "Unclassified" represents disclosed uncertainty, not missing data. The Vessel Type view is the apples-to-apples market-size view; the Hull Program view surfaces the natural limit of classification specificity.

## Policy decisions documented this session

1. **Cross-platform weapon system sustainment is ship MRO** when the weapon is carried by a ship/sub. The weapon's program office (SSP for Trident, NAVSEA for Tomahawk, NAVAIR for Sparrow) does not determine TAM inclusion; the platform does. Aviation-launched weapons stay out because the project scope is ship, not aircraft.

2. **SSBN SHIPALT work is ship MRO** even when issued by Strategic Systems Programs. Physical ship modification to accept a weapon system is ship work.

3. **Weapon PRODUCT procurement is explicitly separate** from Services TAM. Product PSCs (14xx guided missiles, 10xx guns) live on the Product Procurement sheet via `NEWBUILD_PSCS` and don't intersect Services.

4. **Supergroup-only classification is honest**, not a bug. When Tier 3/4/5 or the LLM can only establish supergroup (e.g., "it's a Submarine sustainment contract") without hull specificity, that's the correct output. The Hull Program table's "Unclassified" column shows disclosed uncertainty, not missing data.

5. **Aviation commands are out of Services TAM scope**: NAVAIR, NAWC, Fleet Readiness Center. They constitute a separate Navy aviation MRO market that's not in this model's scope.
